# 阶段里程碑与技术成熟度门槛

> 上位原则见 `00-problem-definition.md`；数据来源与飞轮模型见 `01-data-cost-model.md`。
> 本文件回答：**什么时候可以进入下一阶段？** 核心判据是**底座技术成熟度 + 飞轮启动条件**，不是流程或组织。

## 1. 阶段划分

- `Stage 1`：内环底座建设（0-6 个月）。自采闭环 + 外部接入规范**冻结但不落地**。
- `Stage 2`：底座稳定 + 有限外化（6-18 个月）。1-2 家战略客户受控试点；2nd-dev 生态保持仅接口保留，不公开开放。
- `Stage 3`：飞轮真正启动 + 平台化（18-30 个月）。多源回流成为多样性主力。

阶段切换由三类判据触发：**底座技术成熟度、数据成本下降、有限外化验证**。组织流程不作为切换条件。

**关键原则**：飞轮真正转起来是 Stage 3 的事。Stage 2 的任务是**让飞轮可启动**，不是**让飞轮转起来**——不要把 Stage 3 的指标前置到 Stage 2。

## 2. 门槛优先级（固定不可颠倒）

`底座成本门槛 > 底座技术门槛 > 数据可用性门槛 > 接口就绪门槛 > 产品门槛 > 商业门槛`

上位门槛不满足，下位达标也不得跨阶段。
注：此处不再把"飞轮门槛"作为 Stage 2 的主门槛——飞轮成熟门槛移至 Stage 2 → Stage 3 的准入位置。

## 3. Stage 1 → Stage 2 准入门槛

### 3.1 底座成本门槛（必须满足）

- 自采单位成本连续 2 个季度下降 >= 10%（基准=季度首月）
- 治理自动化率 >= 60%
- 有效采纳率 >= 70%（入池数据被训练消费）

### 3.2 底座技术门槛（必须满足）

- `UDV_total > 0` 且连续 6 周不反转
- 至少 3 类核心任务中，2 类 `UDV_by_task > 0`
- 连续 2 次版本迭代无 P0 回归

### 3.3 数据可用性门槛（必须满足）

- 连续 2 个月有效数据小时稳定增长
- 质检通过率 >= 92%
- 失败样本占比保持目标区间（20%-35%）

### 3.4 接口就绪门槛（必须满足，轻量）

仅要求**规范冻结**，不要求真实开放：

- 外部回流 SDK 规范 V1 冻结并通过内部 Mock 测试
- 数据权属 schema V1 冻结且法务审核通过
- MVP 能力边界文档冻结（可开放 / 不可开放能力清单）

### 3.5 产品化准备门槛（满足 2/3 即可）

- 试点标准验收协议冻结
- 版本发布节奏可预测（过去 6 周按节奏发布 >= 80%）
- 至少 1 套评测规范文档化可对外讲清

## 4. Stage 2 运行目标（非硬门槛，仅过程目标）

Stage 2 是"底座稳定 + 有限外化"的窗口，**不是飞轮启动窗口**。本阶段的运行目标不作为跨阶段硬门槛，仅作为过程验证：

### 4.1 底座持续迭代（主线）

- 内环周级闭环连续运行 >= 20 周不中断
- UDV_total 稳定提升，至少 2 类核心任务 `UDV_by_task` 每季度改善
- 自采单位成本持续下降（每季度 >= 5%）

### 4.2 有限外化试点（副线，规模受控）

- 最多 2 家同场景行业客户付费试点，目标是验证"可交付性"，不是规模营收
- 最多 1-2 家战略伙伴非公开联合验证，验证接口规范可用性
- 外部部署总规模不超过内部门店规模的 30%

### 4.3 Stage 3 门槛前置证据收集（副线）

- 积累跨场景迁移数据，为 Stage 3 跨场景 UDV 评估做准备
- 不设"回流有效小时占比"硬指标——该指标移至 Stage 3 准入门槛

### 4.4 明确不做的事（避免战略漂移）

以下动作在 Stage 2 不启动：

- 2nd-dev 生态公开开放、开发者门户、公开 SDK 发布
- "生态伙伴数"作为 KPI
- 外部回流数据混入主训练集（只走隔离池）
- 承诺跨行业泛化或重度定制能力

## 5. Stage 2 → Stage 3 准入门槛

### 5.1 飞轮启动门槛（必须满足，**本阶段最硬门槛**）

- 底座模型跨场景 / 跨本体 UDV 连续 8 周为正（证明模型可消化异构数据）
- 外部回流有效小时占比 >= 15% 且连续 8 周稳定（从 Stage 2 末期的小规模回流启动）
- 跨场景 UDV 与自有场景 UDV 差值 <= 30%
- Stage 2 期试点客户/伙伴中，至少 2 家产出正向回流证据

注：**若模型无法证明能利用异构数据（第 1 条），即使其他门槛满足也不启动 Stage 3**。这是整个战略最硬的技术卡点。

### 5.2 技术产品门槛（必须满足）

- 跨场景（门店/行业）迁移性能损耗 <= 20%
- 新客户 / 新伙伴上线周期 <= 10 周
- 模型版本回归自动化覆盖 >= 80%

### 5.3 平台能力门槛（满足 2/3 即可）

- DataOps / Compiler / Eval 三模块完成标准化版本
- 数据接口规范或评测规范至少发布 1 套并稳定使用
- 失败模式知识库覆盖核心任务 Top20 异常

### 5.4 商业验证门槛（满足 2/3 即可，不作为主触发）

- 外部 PoC 转付费率 >= 40%
- 外部收入占总成本 >= 10% 且连续 1 季度为正毛利
- 至少 1 套标准接口被合作方规模化接入

### 5.5 Stage 3 储备模块

- 机器人技能数据生态平台（消费级头戴采集 + 技能封装分发）
  - 评估记录：`archive/bp-assessment-skill-ecosystem.md`
  - 采纳方案：降级为 Stage 3 可选模块
  - 重评触发：VLA 第一视角视频到真机策略在受控任务达 >= 60%

## 6. 加速通道（仅窗口期启用）

允许在以下情形提前局部外化：

- 战略客户绑定硬件大单且内部节奏可控
- 监管或生态合作窗口期短

启用条件：

- 3.1 / 3.2 / 3.3 / 3.4 所有硬门槛全部达标
- 仅开放 MVP 能力，不开放定制
- 明确加速期限与回退阈值

## 7. 决策输出（每月一次，产品总监视角）

每评估周期只输出三个结论：

1. `是否跨阶段`：通过 / 延期 / 有条件通过
2. `当前阶段内是否加码外化`：是 / 否 / 维持
3. `补哪些短板`：数据、模型、工程、成本、接口就绪

## 8. 预警与回退（技术触发）

出现以下任一情况，触发回退评估：

- 底座成本门槛连续 2 个季度不达标
- Stage 2 期外部试点占用资源导致内部里程碑延误 > 20%
- Stage 3 启动后跨场景 UDV 连续 2 个迭代反转（证明模型无法利用异构数据）

回退动作：

- 暂停新增外部试点与接入
- 资源优先回补内环闭环
- 4 周内完成修复并复评
