# BP 评估：机器人技能数据生态平台

> 决策记录（已采纳）
>
> - 决策日期：2026-04-19
> - 决策结论：采纳选项 A
> - 处置方式：不立为独立项目，降级为 Stage 3 生态扩展期的可选模块储备
> - 主线影响：不占用 Stage 1 / Stage 2 战略资源池
> - 重评触发条件：第 9 节三件事任一被行业或内部验证达标时，启动重新评估

## 1. 评估对象

- 文档：`商业计划书_机器人数据生态平台.md`
- 核心模式：头戴采集设备 + 云端训练 + 技能商城 + 众包数据市场
- 关键承诺：2026 H1 VLA 模型在 3 类技能成功率 >= 70%；2029 年营收 2-11 亿

## 2. 评估结论（一句话）

方向正确但时机太早，与本司“先内后外、技术门槛优先、零售/服务场景”战略结构性冲突，不建议作为主线立项。

## 3. 技术可行性（最高风险）

BP 自评成熟度：

- VLA 基础模型：学术 3 星 / 工程 2 星
- 跨体态迁移：学术 2 星 / 工程 1 星
- Latent Skill Token：学术 2 星 / 工程 1 星

对应商业化承诺：2026 Q2 达成可商用精度。评估：

- 学术与工程现状远不支持此时间节奏
- 头部产业方（灵初、北京人形、觅蜂）仍在用真机+遥操+无本体混合路径
- 仅用消费级头戴第一视角视频跨越 embodiment gap，缺乏已验证先例

## 4. 商业假设链条风险

核心 5 层嵌套假设，任一断裂即全盘失效：

1. 机器人 2027 年 60,000 台量产成立
2. 20% 机器人用户购买 3000 元配件
3. 大量兼职采集方承担 3000 元入场成本并持续参与
4. 家庭用户为单技能付费 99 元且高频复购
5. VLA 模型可实现少样本稳定提效
6. 数据飞轮正向循环而非成本失控

当前均无前置证据支持。

## 5. 财务模型主要红旗

- 云训练毛利 60% 未摊销 VLA 基础模型研发成本（亿元级）
- 众包毛利 100% 忽略质检、客服、合规、风控成本
- 2029 年 C 端技能交易 142.8 万笔/年无行业先例
- 保守估计下仍需上万台设备与百万级 GMV 支撑，运营成本未充分建模

## 6. 与本司战略冲突点

- 冲突“先内后外”：BP 为全外循环
- 冲突“技术门槛优先”：BP 以商业节奏为门槛锚
- 冲突“零售/服务场景优先”：BP 主打 C 端家庭长尾场景
- 冲突“不做纯平台/轻壁垒”：BP 仍是“硬件+平台”轻资产模式

## 7. 可吸收的有价值部分

- 众包 + 数据飞轮的长期方向正确
- 头戴设备可作为 Stage 3 的无本体采集补充方案
- 技能封装方向（Latent Skill Token + LoRA）值得预研
- 产品分层架构（硬件/平台/商城）可作为远期演进蓝图

## 8. 三选一建议

### 选项 A（建议）

不立为独立项目，降级为本司 Stage 3 生态扩展期的可选模块：

- 消费级辅助采集设备
- 技能封装协议与分发格式

### 选项 B

立为预研项目（R&D-only），6-12 个月内仅验证：

- 第一视角视频到机器人可部署策略在受控 3 类任务上是否可达 >= 60% 成功率
- 不做 C 端、不做众包、不做商城

### 选项 C（不建议）

按 BP 原节奏推进，将稀释本司“本体+零售服务场景闭环”主线资源，中短期 ROI 不可控。

## 9. 继续推进前必须先验证的三件事

1. 价值验证：VLA 第一视角视频是否可在真机达到 >= 60% 成功率（受控 3 类任务）
2. 付费验证：家庭用户对单技能的真实付费与续购行为（小样本实测）
3. 供给验证：兼职采集方在 3000 元门槛下的持续参与率（试点 200-500 人）

三件事任一不过关，财务推演不具备讨论价值。

## 10. 风险矩阵（摘要）

| 风险 | 等级 | 影响 | 推荐动作 |
|---|---|---|---|
| VLA 能力不达成 | 高 | 整条链路失效 | 选项 B 先验证 |
| 跨体态迁移不可解 | 高 | 纯视频路径不可行 | 备选硬件增加姿态传感器 |
| 众包供给不足 | 中 | 数据飞轮停滞 | 先小规模试点 |
| C 端付费不足 | 中 | 收入模型崩塌 | 小样本付费测试 |
| 合规与隐私 | 中 | 法律风险 | 预研期即引入法务评审 |
| 资源稀释主线 | 高 | 主线战略受损 | 严控投入上限与出口条件 |

## 11. 结论性建议

- 本 BP 不进入本司当前 Stage 1 / Stage 2 战略资源池
- 允许以“预研或选项 A”形式保留路径
- 任何升级到主线项目的前提：第 9 节三件事全部验证通过
