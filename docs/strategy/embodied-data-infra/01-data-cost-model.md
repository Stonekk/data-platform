# 数据来源、成本与飞轮模型（核心决策文档）

> 上位原则见 `00-problem-definition.md`。本文件回答三个问题：
> 1) 如何以最低综合成本获得足够高质量的训练数据（内环 / Problem A）
> 2) 如何通过外部部署与生态开放放大数据多样性（外环 / Problem B 的数据价值）
> 3) 外部变现作为次要副产物如何处理（外环的商业价值）

## 1. 决策目标（主目标 + 长期保留目标）

本阶段的决策目标是**单主目标 + 一个保留目标**，不是双目标并行：

- **主目标（Stage 1 / Stage 2 全力投入）**：`MinimizeTotalCost` = Sum_i (Hours_i × UnitCost_i) / ValueDensity_i
- **保留目标（Stage 3 才真正启用）**：`MaximizeFlywheelGain` = Diversity(外部回流) × Scale(外部回流) × 跨场景 UDV

当下阶段 ROI 口径：

`ROI_now = (成本下降节约 + 模型能力价值折算) / 总投入`

当飞轮启动前置条件满足后（见 §4.2），才把飞轮增益并入 ROI 公式。**过早并入会导致战略被长期叙事绑架。**

## 2. UDV：价值密度指标

`UDV = ΔSuccessRate / EffectiveDataHours_added`

- 内环用途：比较自采/外采/治理路径的价值密度。
- 外环用途：度量回流数据是否真的扩展了模型能力边界（`跨场景 UDV`）。
- 有效小时计入条件：完成质检 + 带任务语义标签 + 被训练消费 + 对应模型版本已部署。

## 3. 数据来源五路径对比（TCO + 飞轮框架）

相比纯成本比较，加入"飞轮贡献"列。

| 模式 | 单位成本 | 价值密度 | 多样性贡献 | 起量速度 | 质量可控 | 合规风险 | 战略定位 |
|---|---|---|---|---|---|---|---|
| 自采（真机+自有场景） | 高 | 高 | 低 | 慢 | 高 | 低 | **Stage 1/2 主力** |
| 外采（第三方数据服务商） | 中 | 中 | 中 | 快 | 中 | 中 | **Stage 1/2 补盲** |
| 生态（云/硬件伙伴共建） | 低-中 | 中-高 | 中-高 | 中 | 中 | 中 | Stage 2 末期试点 |
| 众包（消费级设备+C端） | 低 | 低 | 高 | 慢-中 | 低 | 高 | Stage 3 潜力，前期不投 |
| 2nd-dev 生态开放 | 低（边际成本） | 视伙伴场景 | 高（跨行业/跨本体） | 中 | 需架构保障 | 中 | **Stage 3 主引擎，Stage 1/2 仅留接口** |

### 选型原则

- 起步期（Stage 1）：自采 + 外采为主；同时冻结对外回流 SDK 与数据权属 schema（便宜保险，不落地）。
- 爬坡期（Stage 2）：仍以自采 + 外采为主；可对 1-2 家战略生态伙伴做受控试点，但不作为主要数据来源。
- 规模期（Stage 3）：2nd-dev 生态与众包才成为多样性主力；此时飞轮真正转起来。

**反模式提示**：把"2nd-dev 生态开放"当作 Stage 2 的数据来源主力，是一种常见的战略漂移——看似激进进取，实际是把未经验证的长期赌注当成短期确定性来源，底座团队会被外部接口维护与合作方支持稀释。

## 4. 数据飞轮模型（长期形态，非当前执行项）

### 4.1 飞轮的四个节点

1. **自采** → 自有场景数据 → 训练底座模型
2. **外化** → 模型能力部署到客户 / 伙伴 / 2nd-dev 开发者
3. **回流** → 客户与伙伴场景的新数据（ego 视频、跨本体、跨行业长尾）回到治理层
4. **再训练** → 模型能力边界扩展，下一轮外化更广

### 4.2 飞轮启动的真实前置条件（全部满足才启动）

1. **技术条件**：底座模型在跨场景 / 跨本体 / 跨视角迁移上有可量化的能力证据（非当前学术 SOTA 的水平）。
2. **工程条件**：内环周级闭环稳定 ≥ 12 周（见 `03-data-infra-v1.md` 第 7 节）。
3. **接口条件**：外部回流 SDK 与数据权属 schema 已冻结并通过内外审计。
4. **合规条件**：数据脱敏、跨境、伙伴授权条款完整。

**当前（2026 年）这 4 个条件中，条件 1 是最硬的卡点，也是行业共性难题。**

### 4.3 飞轮失速预警（启动后监控）

- 外部回流数据的跨场景 UDV 持续为负 → 模型无法利用新多样性 → 回流只是成本负担
- 回流数据采纳率 < 50% → 接入架构或数据质量有结构性问题

### 4.4 为什么不能提前启动

- 底座不稳时开放接口 → 接口维护成本稀释内环资源；
- 底座平庸时开放能力 → 把自身能力短板暴露给合作方和竞品；
- 数据治理未自动化时接入外部 → 治理成本可能高于自采；
- 伙伴场景分布未知时承诺 SLA → 产品口碑风险。

结论：**飞轮的魅力是长期，代价是短期**。Stage 1/2 只做"便宜保留"，不做"真实启动"。

## 5. 单位成本拆解与降本路径

`UnitCost = (设备摊销 + 人力 + 场地 + 治理 + 算力分摊 + 合规) / 有效小时`

降本优先级（按 ROI 排序）：

1. 治理自动化（质检/对齐/切片）——短期 ROI 最高
2. 采集标准化工具链——中期降边际成本
3. 多源融合（自采+外采+生态）——中期结构性降本
4. 飞轮外环放量——**长期潜力**，边际成本趋零，但要求底座模型能利用异构数据
5. 众包试点（条件受控）——长期潜力但高不确定性

## 6. 假设清单

| ID | 假设 | 当前值 | 目标值 | 证据来源 |
|---|---|---|---|---|
| H1 | 自采单位成本 | 待填 | 按季度下降 >=10% | 内部场景运营 |
| H2 | 外采单位成本 | 待填 | 低于自采 30% | 外部报价样本 |
| H3 | 众包单位成本 | 待填 | 低于自采 50% 且 UDV > 阈值 | 受控试点 |
| H4 | 生态共建成本 | 待填 | 通过共建降摊销 | 合作协议 |
| H5 | 治理自动化率 | 待填 | >=70% | 工具链指标 |
| H6 | 2nd-dev 生态回流多样性 | 待填 | 外部场景数 >= 自有 2 倍 | 生态接入日志 |
| H7 | 跨场景 UDV | 待填 | 不低于自有场景 UDV 的 70% | 模型评测 |

## 7. 外环路径一：对外付费部署（Stage 2 窗口）

- 定位：向同场景行业客户交付"模型 + 数据闭环"解决方案。
- 当前首要产出：行业标杆 + 回流通道验证（不是规模化营收，也不是主数据源）。
- 时机：内环达 Stage 1→2 所有硬门槛后启动（见 `02-stage-milestones.md` §3）。
- 规模上限：Stage 2 期外部部署数量 ≤ 2 家，避免底座团队被稀释。
- 详细执行参考：`archive/external-pilot-playbook.md`（激活时再启用）。

## 8. 外环路径二：2nd-dev 生态开放（Stage 3 主引擎）

### 8.1 当前阶段（Stage 1 / Stage 2）只做什么

**只做便宜保留，不做真实开放**：

- Stage 1 末：冻结接口规范与数据权属 schema（见 `03-data-infra-v1.md` §6）
- Stage 2 全程：仅对 1-2 家战略伙伴做**非公开的受控联合验证**，不对任何外部开发者开放

不做什么：

- 不做公开 API 文档、不做开发者门户、不做 SDK 公开发布
- 不承接 2nd-dev 伙伴的技术支持、不投入 DevRel 人力
- 不以"生态伙伴数"作为任何团队的 KPI

### 8.2 Stage 3 启动条件（必须全部满足）

1. 内环跨场景 UDV 连续 8 周为正
2. 底座模型能可量化地利用异构回流数据（跨本体 UDV ≥ 自有场景 UDV × 0.7）
3. DataOps / Compiler / Eval 完成标准化版本
4. 至少 2 家 Stage 2 伙伴的受控验证产出正向回流证据

### 8.3 Stage 3 开放对象（分层）

按接入门槛从低到高，**但进入顺序由 Stage 3 启动后再决策**：

- 模型能力 API / 技能调用接口（黑盒能力）
- 数据接入 SDK（允许伙伴把场景数据回流到平台）
- 评测与失败模式库（让伙伴自助诊断）
- 精调与 LoRA 分发（让伙伴在平台上训练特化能力）

### 8.4 开放边界（避免战略漂移）

- 不对外开放底座训练数据原始权属
- 不承诺跨行业泛化与重度定制效果
- 外部收入占总成本上限：Stage 2 ≤ 10%，Stage 3 ≤ 50%

### 8.5 为什么不能在 Stage 2 提前开放（技术判断）

- 学界 SOTA 的跨本体/跨视角迁移效果仍不稳定，强行开放等于把能力短板公开；
- 平台生态网络效应依赖"核心能力显著领先"，咱们现在不是这样；
- API 和 SDK 的向后兼容一旦承诺，重构成本极高，不宜过早冻结。

## 9. 外环路径三：工具链 / 标准授权（长线）

- DataOps / Compiler / Eval 标准化后对行业授权。
- 仅在 Stage 3 开启，不占用 Stage 1-2 资源。
- 价值偏行业位势，直接营收不作为主指标。
