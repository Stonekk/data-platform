# 问题定义与上位原则

## 1. 背景

本司已确定“物理 AI 大模型”为既定战略，因此训练所需的高质量、高多样性交互数据是**刚性需求**。
战略核心不是“要不要投入数据”，而是**用什么模式以最低综合成本、最快速度获得足够多样且足够有价值的数据**。

## 2. 上位机制：数据飞轮（长期战略，不是短期门槛）

数据基础设施的长期形态是一个**双环耦合飞轮**：

```
内环（能力建设，当前阶段重心）     外环（能力外化，长期目标）
自采数据  →  模型能力  →  对外部署 / 生态开放
   ↑                              ↓
   └─  回流 ego / 本体 / 视频 / 长尾  ──┘
             （多样性与规模）
```

- **内环**：自有场景自采 → 训练 → 模型能力。解决"从 0 到 1"。**这是 Stage 1 / Stage 2 的主战场。**
- **外环**：对外部署与生态开放 → 回流跨场景 / 跨本体 / ego 数据 → 扩大多样性。解决"从 1 到 N"。**这是 Stage 3 才真正转起来的引擎。**

### 关于飞轮的关键判断（避免误用叙事）

飞轮能转有两个技术前置条件当下都未满足：

1. 底座模型能稳定消化异构来源数据（跨场景 / 跨本体 / 跨视角）；
2. 外部部署的规模和数据权属治理能支撑回流。

在这两个条件满足前，**外环不是"主引擎"，只是"保留选项"**。过早把飞轮当主路径，会演变为"用长期叙事绑架短期执行"。

因此本战略的真实打法是：

- **当下（Stage 1 / Stage 2）**：全力打磨内环，外环只做 schema 与接口的**便宜保留**（不落地）。
- **中期（Stage 2 末 / Stage 3 初）**：底座技术与数据权属成熟后，才启动外环真实开放。
- **长期（Stage 3 及以后）**：飞轮成为第二增长曲线。

## 3. 两类问题（主线不变）

### Problem A：数据底座建设（内环，主线）

- 目标：在自有场景完成从采集到模型迭代的周级闭环，单位有效数据成本持续下降。
- 关键杠杆：自采 / 外采 / 治理自动化 / 增量训练。
- 衡量指标：`单位有效数据成本`、`数据采纳率`、`质检通过率`、`UDV（价值密度）`。
- **当前阶段资源分配 >= 80%。**

### Problem B：飞轮外化与生态开放（外环，长期战略，当前只做保留）

- 当前阶段目标：冻结接口 / 数据权属 / 评测规范 schema，**不做真实对外开放**。
- 中长期目标：当 A 达标且跨场景 UDV 可验证后，启动真实外化，获取多样性回流。
- 衡量指标（未来启用）：`回流多样性`、`跨场景 UDV`、`回流有效小时占比`。
- **当前阶段资源分配 <= 10%（仅 schema 与规范类工作）。**

### Problem C：模型与本体商业化（下游，不在本文档范畴）

- 机器人本体销售、服务、场景运营等业务层变现不在本战略域内讨论。

## 4. 决策原则

1. **A 是主线，B 是保留**。A 的每周级稳定度是任何 B 动作的前提。
2. **飞轮是长期战略叙事**，不作为 Stage 2 的硬门槛。Stage 2 只负责让"飞轮可以启动"，不负责让"飞轮转起来"。
3. 任何加速 B 的提议必须先回答：
   - 是否延误 A 的周级闭环？
   - 底座模型是否已证明能利用异构回流数据？
4. 任何新采集/接入模式必须先回答：是否降低 A 的单位成本，或为未来 B 的启动降低切换成本。
5. 生态开放（2nd-dev / 商业伙伴）的节奏：Stage 1 末冻结 schema → Stage 2 仅对 1-2 家战略伙伴试点 → Stage 3 真正规模化。

## 6. 上位判断（给决策者看的四句话）

- 数据是确定性战略投入，不是商业赌注。
- 当下（12-18 个月内）核心 ROI = 单位数据成本下降 × 模型价值密度。
- 飞轮是 24-36 个月的长期引擎，不是 Stage 2 的执行门槛；Stage 2 只需把"飞轮可以启动"的接口和规范准备好。
- 飞轮真正转起来的前提是：底座模型能消化异构数据。这个技术条件没达到前，所有外环加速都是空中楼阁。
