# V2.3.0 迭代说明

> 开启日期：2026-06-18  
> 基线：V2.2.0（台本生成引擎 + DeepSeek + 冷启动数据 + 场景×动作大类）

## 版本定位

- **版本号**：`2.3.0`
- **主题**：台本生成**生产化**与**可控变体**——从「能生成」到「可运营、可泛化、可部署」
- **性质**：前端原型能力迭代；延续 mock 数据 + 本地/代理 LLM，不引入完整后端

## V2.3.0 目标（本迭代）

### P0 — 台本优先业务流（V2.3.0 核心）

- [x] **批量台本候选**：需求画像 → 一键生成 10/30/50 条候选
- [x] **运营确认与采集次数**：候选池筛选、确认、每条台本配置目标条数
- [x] **多台本任务单元**：`Task.scriptIds` + `scriptAllocations`，按场景+道具+类型分组
- [x] **下游兼容**：就绪校验、任务列表、自动调度、采集 App 任务包展示
- [x] **规格与验证**：`docs/superpowers/specs/2026-06-18-v230-script-first-workflow-design.md` + `scripts/validate-script-batch-workflow.mjs`

### P0 — 生成质量与运营闭环（延续）

- [ ] **L2 LLM 软判**：批量候选合理性评审（边缘常识 case），与 L1 串联
- [ ] **道具功能角色**：`Prop.role` 或等价字段，支撑 filter / Prompt 约束
- [x] **台本确认后追溯**：`generationMeta` / `batchId` / `reviewStatus` 写入 `TaskScript`（演示级）
- [x] **L1 质检 UX**：`ScriptQcBadge`、候选按评分排序、未过质检醒目展示与一键剔除

### P1 — 可控变体（设计规格 P2）

- [ ] **`VariationSampler`**：四维变体卡池（手部/空间/物理/静态感知）受控采样
- [ ] **`StaticPerceptionProfile`**：perception 场景光照/杂乱/反光配置 UI
- [ ] **T-001-C few-shot**：无模板 + perception 场景走 LLM 路径时注入黄金样例
- [ ] **模板迁移**：`tpl-002` ~ `tpl-004` 四槽 skeleton

### P2 — 生产与部署

- [ ] **生产 LLM 通路**：`VITE_LLM_PROXY_URL` 或静态站可调用的转发说明/配置
- [ ] **Cos / Vercel 部署验证**：台本生成在目标环境 smoke test

### P3 — 体验打磨（有余力）

- [ ] 动作大类 Chip 展开原语预览（Popover，只读）
- [ ] 原子动作库搜索跨场景（管理页）
- [ ] 采集 App 展示 `renderedCard` 与录制时长摘要

## 不在本迭代

- 动态感知干扰（人影、电视闪烁）
- 端侧 YOLO 盲盒匹配
- 平台母任务编号体系
- 原语级勾选（维持「场景×动作大类」粒度）

## 验收口径（迭代完成时）

1. perception 场景可生成 T-001-C 类变体，manipulation 场景仍拒液体/明火。
2. 同一母任务可产出 ≥3 个**物理意义不同**的变体（非仅改措辞）。
3. 生产构建下 LLM 模式可配置并成功生成（非仅 dev 代理）。
4. 运营可在台本确认页看到生成来源与筛选统计。
