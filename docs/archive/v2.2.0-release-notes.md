# V2.2.0 Release Notes

> 发布日期：2026-06-18  
> 基线：V2.1.0（任务台本 / 排期 / 采集端原型）

## 版本定位

- **版本号**：`2.2.0`
- **主题**：参数化任务台本生成引擎 + DeepSeek LLM 集成
- **性质**：前端原型能力升级；LLM 调用经 Vite dev 代理，生产需自建 OpenAI 兼容后端

## 亮点

### 1. 任务台本生成引擎（`src/lib/scriptGenerator/`）

- **四槽骨架**：导读 / 环境场控 / 道具摆位 / 动作序列 + 防废片红线
- **三种 Provider 模式**（`VITE_SCRIPT_GENERATOR_MODE`）：
  - `template` — 规则模板填槽
  - `mock-llm` — 模拟批量候选 + 筛选（离线调试）
  - `llm` — DeepSeek API 真实生成
- **路径 C**：有 `ScriptTemplate.skeleton` 时模板优先；否则 LLM 自由生成 + few-shot
- **输出**：结构化 `TaskScriptCard` JSON + 确定性渲染 `renderTaskCard()` → 采集员可读卡片

### 2. DeepSeek API 集成

- 本地开发：`vite.config.ts` 代理 `/api/llm` → DeepSeek OpenAI 兼容接口
- 配置：`.env.example` 提供 `DEEPSEEK_API_KEY`、`VITE_SCRIPT_GENERATOR_MODE=llm`
- 任务配置面板（`ScriptConfigPanel`）一键生成候选台本，展示通过/筛除统计

### 3. L1 硬规则筛选（`filter.ts`）

| 规则 | 拒因示例 |
|------|----------|
| 全局安全 | 明火、触电等 |
| `safetyTier` | manipulation 场景禁液体 |
| 道具白名单 | 使用了未配置道具 |
| 难度匹配 | 需求 complex 混入 correction |
| 纠错档 | 无失误/恢复描述 |
| **常识合理性** | 玻璃杯放入保鲜盒等荒谬组合 |

### 4. Skill 与验证工具

- `.cursor/skills/task-script-generator/` — 生成规范、黄金样例、自检清单
- `docs/superpowers/specs/2026-06-12-task-script-generator-design.md` — 设计规格
- `scripts/validate-task-script-skill.mjs` — 可选 GPT/ fixtures 验证 Skill 合规性

### 5. 数据与 UI

- 冷启动种子：`src/data/seed/`（场景、道具、场景-动作索引）
- 原子动作库重构：`atomicActions.ts` + 管理页展示细粒度原语
- 台本卡片 UI：`ScriptContentCard` 支持四槽结构化展示

## 已知限制

- L2 LLM 软判（批量合理性评审）尚未接入，边缘常识 case 仍依赖 Prompt + 有限正则
- 生产静态部署无 dev 代理，需配置 `VITE_LLM_PROXY_URL` 或后端转发
- 道具库暂无「功能角色」字段，合理性规则以 Skill 描述 + 硬编码模式为主

## 如何运行

```bash
cp .env.example .env.local
# 填入 DEEPSEEK_API_KEY，设置 VITE_SCRIPT_GENERATOR_MODE=llm

npm ci
npm run dev
```

在任务详情 → 台本配置 → 生成台本。

git checkout v2.2.0
