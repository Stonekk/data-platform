# 具身数据平台 · 前端原型 · 研发本地运行

本仓库为 **Vite + React 19 + TypeScript** 单页应用原型，默认 **不依赖后端服务** 即可本地预览界面与交互（数据以页面内 mock / 本地状态为主）。

## 对方电脑需要什么

- **Node.js**：建议 **20 LTS** 或 **22**（[nodejs.org](https://nodejs.org/)）
- **npm**：随 Node 安装即可（本仓库含 `package-lock.json`，推荐用 `npm ci` 锁定依赖版本）

## 怎么发给他（任选其一）

### 方式 A：压缩包（适合不开放 Git 权限时）

1. 在本机项目根目录执行（已提供脚本）：

   ```bash
   chmod +x prototype-handoff/打包源码给研发.sh
   ./prototype-handoff/打包源码给研发.sh
   ```

2. 脚本会在**项目上一级目录**生成 `embodied-data-platform-prototype-YYYYMMDD-HHMMSS.tar.gz`。
3. 把该 `.tar.gz` 通过企业网盘 / IM / 邮件附件发给对方；对方解压后进入解压出的文件夹，按下面「本地启动」操作。

> 若手动打包：请**不要**打进 `node_modules/`、`dist/`，可省略 `.git/` 以减小体积；务必保留 `package.json` 与 `package-lock.json`。

### 方式 B：Git 仓库（适合已有代码托管）

1. 给对方 **只读仓库地址** + **分支名 / tag**（例如 `main` 或 `v2.0.0-prototype`）。
2. 对方执行：

   ```bash
   git clone <仓库URL>
   cd embodied-data-platform
   git checkout <分支或tag>
   ```

3. 然后按下面「本地启动」操作。

## 对方本地启动（解压或 clone 之后）

在**项目根目录**（与 `package.json` 同级）执行：

```bash
npm ci
npm run dev
```

浏览器打开终端里提示的地址，一般为：

`http://localhost:5173/`

停止服务：在终端按 `Ctrl+C`。

### 可选：生产构建预览

```bash
npm run build
npm run preview
```

## 常见问题

- **`npm ci` 报错**：若对方没有 `package-lock.json`（例如只拷了部分文件），可改用 `npm install`。
- **端口被占用**：Vite 会自动尝试下一个端口，以终端输出为准；也可在项目根执行 `npx vite --port 5174` 指定端口。
