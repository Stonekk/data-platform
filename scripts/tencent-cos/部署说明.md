# 腾讯云 COS 部署（具身数据平台前端）

适用：**已注册腾讯云**，用 **对象存储 COS + 静态网站** 托管 `npm run build` 产物（与 Vercel 同类，国内访问更稳定）。

## 一、控制台准备（只需配置一次）

1. 打开 [对象存储 COS 控制台](https://console.cloud.tencent.com/cos)，**新建存储桶**  
   - 建议选离你用户近的**地域**（如广州 `ap-guangzhou`）。  
   - 记下完整桶名，形如 **`yourname-1250000000`**（含 APPID）。

2. 进入该桶 → **权限管理**  
   - **公有读私有写**（或「公有读」策略）：让浏览器能直接 GET 静态文件。仅原型/内网可接受；生产请按公司规范收紧（如仅 CDN 回源、鉴权等）。

3. 同一桶内 → **基础配置** → **静态网站** → **编辑**  
   - **开启**静态网站。  
   - **索引文档**：`index.html`  
   - **错误文档**：填 **`index.html`**（本项目使用 React Router，必须让 404 也返回首页，否则刷新子路径会白屏。）

4. 保存后，页面会给出 **静态网站访问节点**，形如：  
   `http://<BucketName-APPID>.cos-website.<Region>.myqcloud.com`  
   这是默认的 **HTTP** 访问地址。需要 **HTTPS** 时，在控制台为该桶配置 **自定义域名 + CDN**（或绑定已备案域名并按文档开启 HTTPS）。

## 二、本机安装 COSCLI

官方说明：[COSCLI 工具](https://cloud.tencent.com/document/product/436/63144)

- 下载对应系统的 `coscli`，解压后把可执行文件加入 `PATH`。  
- 首次使用建议执行一次（按提示输入 SecretId / SecretKey，可在 [访问管理](https://console.cloud.tencent.com/cam/capi) 创建子用户密钥并只授权 COS 最小权限）：

```bash
coscli config init
```

若不想写全局配置，可在环境变量里提供密钥（见下）。

## 三、上传代码（每次发版）

在项目根目录：

```bash
export COS_BUCKET=你的桶名-1250000000
export COS_REGION=ap-guangzhou
# 若未执行 coscli config init，可再导出：
# export COS_SECRET_ID=...
# export COS_SECRET_KEY=...

npm run deploy:cos
```

或复制 `scripts/tencent-cos/env.example` 为项目根目录的 `.env.cos`（**不要提交**），填写变量后：

```bash
set -a && source .env.cos && set +a && npm run deploy:cos
```

完成后用控制台里的 **静态网站访问节点**（或你的自定义域名）访问即可。

## 四、常见问题

| 现象 | 处理 |
|------|------|
| 首页能开，刷新子路由 404 | 错误文档未设为 `index.html`，回到「静态网站」检查。 |
| 403 / AccessDenied | 桶或对象的**公有读**未放开，或策略禁止了 `GetObject`。 |
| 只有 HTTP、需要 HTTPS | 绑定自定义域名并走 **CDN** 或官方文档中的 HTTPS 方案。 |
| coscli 报鉴权失败 | 检查 SecretId/SecretKey、子账号策略是否包含该桶；`-e` 地域是否与桶一致。 |

## 五、与 Vercel 的关系

Vercel 与 COS 互不影响；国内分享链接优先发 **COS / 自定义域名**，境外仍可继续用 Vercel（若可访问）。
